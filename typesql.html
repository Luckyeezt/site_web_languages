Les Types de Données en SQL :

Le type de données d'une colonne définit la valeur que la colonne peut contenir : entier, caractère, argent, date et heure, binaire, etc.

  • Les nombres : Représentent les valeurs numériques, tels que les entiers et les décimaux.
    1. Les types de nombres décimaux : représentent les nombres à virgules
    2. Les types de nombres entiers : représentent les nombres normaux, sans virgules
  • Les textes : Représentent les chaînes de caractères, permettant de stocker et manipuler du texte.
  • Les types temporels : Représentent les données liées au temps et aux dates, comme les timestamps et les dates.

Les types de données : Nombres

-> Pour les nombres entiers :

Type de données pour les nombres ENTIERS : Description :
TINYINT(size) Donnée qui comprendra un tout petit nombre entier, qui peut commencer de -127 à 128, le paramètre (size) n'a pas d'effet sur la plage de valeurs autorisées. Il spécifie simplement le nombre de chiffres à afficher lors de l'affichage de la valeur.
SMALLINT(size) Donnée qui comprendra un petit nombre entier, qui peut commencer de -32 768 à 32 767, le paramètre (size) n'a pas d'effet sur la plage de valeurs autorisées. Il spécifie simplement le nombre de chiffres à afficher lors de l'affichage de la valeur.
MEDIUMINT(size) Donnée qui comprendra un nombre entier moyen, qui peut commencer de -8 388 608 à 8 388 607 le paramètre (size) n'a pas d'effet sur la plage de valeurs autorisées. Il spécifie simplement le nombre de chiffres à afficher lors de l'affichage de la valeur.
INT(size) Donnée qui comprendra un nombre entier, qui peut commencer de -2 147 483 648 à 2 147 483 647 le paramètre (size) n'a pas d'effet sur la plage de valeurs autorisées. Il spécifie simplement le nombre de chiffres à afficher lors de l'affichage de la valeur.
BIGINT(size) Donnée qui comprendra un grand nombre entier, qui peut commencer de -9 223 372 036 854 775 808 à 9 223 372 036 854 775 807 le paramètre (size) n'a pas d'effet sur la plage de valeurs autorisées. Il spécifie simplement le nombre de chiffres à afficher lors de l'affichage de la valeur.


-> Pour les nombres décimaux :

Type de données pour les nombres décimaux : Description :
DECIMAL(size , d) Donnée qui représente un nombre décimal à virgule fixe. La taille spécifiée (size) n'affecte pas la plage de valeurs autorisées, mais elle détermine le nombre total de chiffres pouvant être stockés, y compris ceux à gauche et à droite de la virgule. Le paramètre 'd' spécifie le nombre de chiffres à droite de la virgule.
FLOAT Donnée qui représente un nombre décimal à virgule flottante simple précision. Il peut stocker des valeurs avec une précision inférieure par rapport à DECIMAL, mais il utilise moins d'espace de stockage.
DOUBLE Donnée qui représente un nombre décimal à virgule flottante double précision. Il offre une plus grande précision que FLOAT, mais il utilise également plus d'espace de stockage.

Les types de données : Textes

Les textes peuvent contenir des lettres, des nombres, des symboles & et des caractères spéciaux.

Type de données pour TEXTES: Description :
CHAR(size) Donnée qui comprendra une chaîne de caractères FIXE pouvant aller de 0 à 255 caractères. Le paramètre (size) définit la longueur maximale de caractères qui peuvent être dans la donnée.
VARCHAR(size) Donnée qui comprendra une chaine de caractères dont on a défini la limite, pouvant aller de 0 à 65 535 caractères. Le paramètre (size) définit la longueur maximale de caractères pouvant être dans la donnée.
TINYTEXT Donnée qui comprendra une chaine de caractères dont la limite est de 255 caractères.
TEXT Donnée qui comprendra une chaine de caractères dont la limite est de 65 535 caractères.
MEDIUMTEXT Donnée qui comprendra une chaine de caractères dont la limite est de 16 777 215 caractères.
LONGTEXT Donnée qui comprendra une chaine de caractères dont la limite est de 4 294 967 295 caractères.

Les types de données : Temporels

Type de données pour TEMPS: Description :
DATETIME Donnée qui stockera une date dont le format est de "YYYY-MM-DD HH:MM:SS" ("Année-Mois-Jour / Heures:Minutes:Secondes")
DATE Donnée qui stockera une date sous la forme : jour,mois,année ("YYYY-MM-DD")
TIME Donnée qui stockera une heure précise : heure, minutes, secondes ("HH:MM:SS")
YEAR Donnée qui stockera une année seulement entre 1901 et 2155
TIMESTAMP Un horodatage. Les valeurs TIMESTAMP sont stockées sous forme de nombre de secondes depuis l'époque Unix ('1970-01-01 00:00:00' UTC). Format : AAAA-MM-JJ hh:mm:ss. La plage prise en charge va de « 1970-01-01 00:00:01 » UTC à « 2038-01-09 03:14:07 » UTC.
